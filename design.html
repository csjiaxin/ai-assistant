AI Agent Design

Slack Support AI Agent

An intelligent agent that learns from historical Slack thread discussions and generates accurate, context-aware answers to new channel questions.

🤖 RAG Architecture 💬 Slack Integration 🔍 Semantic Search 🧠 Vector Embeddings 🐍 Python 📦 OpenAI / Llama
Overview

Problem & Solution

Problem: In busy Slack channels (e.g. #disturbed), the same questions get asked repeatedly. Valuable answers buried in threads are never surfaced. Engineers waste time giving duplicate answers.
Solution: An AI agent that ingests all historical Slack thread Q&A, encodes them as semantic vectors, and — when a new question arrives — retrieves the most relevant past discussions as context for an LLM to generate a high-quality, grounded answer.
📚

Historical Learning

Continuously indexes past questions, answers, and thread reactions from Slack history into a searchable vector store.

🔍

Semantic Retrieval

Uses embedding similarity (cosine) to find the top-K most relevant historical discussions for any new question.

🧠

LLM Generation

Passes retrieved context + question to an LLM (GPT-4o / Llama 3) to synthesize a grounded, accurate answer.

⚡

Real-time Response

Listens for new messages via Slack Bolt / Event API and posts threaded answers within seconds.

Workflow

Step-by-Step Inference Flow

1

🎤 Question Arrives in Slack

A user posts a question in #disturbed (or any monitored channel). The Slack Bolt app receives the message event via Event API webhook. Questions can also be triggered by @bot-mention.

2

⚡ Cache Check

Redis cache is checked for an identical or very similar recent query. If a cache hit is found (within TTL), the cached answer is returned immediately — saving LLM cost and latency.

3

🔢 Embed the Question

The question text is passed to the embedding model (OpenAI text-embedding-3-small or a local sentence-transformer). A 1536-dimensional float vector is produced representing the semantic meaning of the question.

4

🔍 ANN Vector Search (Top-K Retrieval)

The query vector is used to search the vector store (Chroma / Pinecone / Qdrant) using approximate nearest-neighbor (HNSW index). The top 5 most semantically similar historical Q&A threads are returned along with their similarity scores and metadata.

5

📝 Prompt Construction

A structured prompt is built containing: system instructions (persona, constraints), retrieved context (top-K Q&A pairs with scores and source links), and the new question. The LLM is instructed to cite sources and acknowledge uncertainty.

6

🧠 LLM Generation

The prompt is sent to the LLM (GPT-4o via OpenAI API or self-hosted Llama 3 via Ollama). The model synthesizes a grounded answer using the retrieved context, adding its own reasoning where appropriate. Streaming is used for fast time-to-first-token.

7

✅ Post Answer to Thread

The generated answer is posted as a threaded reply in Slack. Source links (deep links to original Slack messages) are appended. The answer also gets reaction buttons (👍/👎) for feedback collection. The response is cached in Redis.

Prompt Engineering

LLM Prompt Template

# System Prompt SYSTEM = """ You are a helpful AI support assistant for a software engineering team. You have access to historical Q&A discussions from our Slack channels. Your job is to answer the new question using the provided context. Rules: - Base your answer primarily on the provided context - Cite relevant sources using [Source N] notation - If context is insufficient, say so clearly — do not hallucinate - Keep answers concise but complete - Use markdown formatting for code snippets and lists """ # Context Injection (top-K retrieved Q&As) CONTEXT_TEMPLATE = """ --- Historical Context --- {%- for i, item in enumerate(retrieved_items) %} [Source {{ i+1 }}] (similarity: {{ item.score:.2f }}, 👍 {{ item.reactions }}) Thread: {{ item.slack_url }} Q: {{ item.question }} A: {{ item.answer }} {%- endfor %} --- End Context --- """ # Final User Message USER_MSG = """ New Question: {{ question }} Please answer using the context above. Cite sources where relevant. """

Tech Stack

Technology Choices

🐍

Python 3.11+

Core language. Type-annotated throughout. No heavy framework needed.

💬

Slack Bolt

Official Slack SDK for Python. Socket Mode — no public URL required.

🤗

sentence-transformers

Free HuggingFace models. Local CPU/GPU inference. No API key. Models: MiniLM, BGE, MPNet.

🦙

Ollama (local LLM)

Free local inference server. Runs Llama 3, Mistral, Gemma, Phi3 on your machine via plain HTTP.

🗄️

ChromaDB

Open-source embedded vector database. HNSW index. Zero infrastructure, persistent on disk.

⚡

Redis (optional)

Cache for repeated queries. Agent works without it — caching gracefully disabled if unavailable.

🐳

Docker + Compose

Containerized deployment with Redis, Prometheus, Grafana included.

📊

Prometheus/Grafana

Optional metrics and dashboards for operational visibility.
